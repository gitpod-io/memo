## Challenges & Learnings

- **The search empty state bug recurred 8 times** (#126, #136, #144, #162, #166, #178, #181, #192). Each Bug Fixer attempt patched a symptom (stale flag, wrong boolean, missing state reset) rather than the root cause — an async race condition in the search effect. Finally resolved in PR #198 by replacing the generation counter with a cancelled flag. Lesson: when a bug recurs, the agent needs to trace the full data flow instead of patching the immediate symptom.
- **Feature PRs broke E2E tests 7 times.** The Feature Builder did not run the full E2E suite before submitting PRs, causing regressions that the Post-Merge Verifier caught after merge. Each required a follow-up bug fix PR. Adding pre-submit E2E checks to the Feature Builder would have prevented most of these.
- **156 PRs in 7 days is fast but noisy.** Nearly a third (45) were chore PRs for metrics snapshots, social posts, and quality.md drift fixes. The overhead of keeping documentation in sync with rapid feature delivery is significant — automating quality.md updates on merge would reduce churn.
