# Week 26 Recap — June 15–21, 2026

Tenth week. A quiet week focused on accessibility, navigation quality, and noise reduction. Three feature PRs shipped — all addressing UX gaps rather than new surfaces. The remaining open issues continue to wait for human approval, keeping the automation pipeline idle between bursts.

## Highlights

- **Proper link navigation across the app.** Sidebar page tree, workspace home lists, and favorites now use `<Link>` instead of `<button>` + `router.push()`, restoring middle-click, right-click "Open in new tab", and browser prefetching. PR #1390.
- **Settings tab navigation.** A shared tab bar (General | Members) replaces the one-way text link, making settings sub-page navigation symmetric. PR #1391.
- **ARIA live regions on all app-side forms.** Screen readers now announce error and success messages in 8 form components that were previously silent. PR #1374.
- **Browser extension noise filtered from Sentry.** MetaMask and other extension errors no longer pollute error tracking. PR #1401.

## Features Shipped

### Navigation

- **Next.js Link for page navigation** — sidebar page tree, workspace home "All Pages", "Recently Visited", and favorites now use `<Link>` elements. Middle-click, right-click context menu, status bar URL preview, and automatic prefetching all work. PR #1390 (Issue #1388).

### Accessibility

- **ARIA live region roles on forms** — added `role="alert"` to error messages and `role="status"` to success messages across 8 app-side form components. Added `aria-hidden="true"` to 4 hidden file inputs. PR #1374 (Issue #1372).

### Error Tracking

## Challenges & Learnings

- **The approval bottleneck persists.** Same pattern as W25: all 4 feature issues shipped within hours of being unblocked, then the pipeline went idle. The 6 remaining open issues have been waiting 22–25 days for human approval. The agents have capacity but nothing to work on.
- **Navigation refactors touch many test selectors.** Replacing `<button>` with `<Link>` in PR #1390 required updating selectors in 6 E2E test files — every test that clicked a page item needed `button` → `a` selector changes. This is a recurring cost of improving semantic HTML after the fact; getting element choices right the first time avoids downstream test churn.
