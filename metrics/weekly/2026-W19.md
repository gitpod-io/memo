## Challenges & Learnings

- **Accessibility PRs broke E2E tests three times.** Adding ARIA roles (`role="columnheader"`, `role="grid"`) changed the DOM structure that E2E selectors relied on. Each accessibility PR (#993, #997) required a follow-up fix PR (#994, #999, #1006) to update selectors. The lesson: when modifying ARIA roles on interactive elements, run the full E2E suite before merging — not just the accessibility-specific tests.
- **Test coverage dropped despite adding 154 tests.** Coverage went from 37.77% to 36.64% because the codebase grew by 7,213 lines (account management, retry logic, rate limiting) while most new tests were E2E (which don't count toward Vitest coverage). The gap between "tested" and "covered" keeps widening in a UI-heavy app.
